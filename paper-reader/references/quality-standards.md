# 笔记质量详细规范（生物医学方向）

## 核心原则

笔记是研究工具，不是摘要翻译。每个字段都要**直接服务于读者判断这篇论文值不值得细读**。

---

## 必检清单（每篇笔记生成后验证）

### 研究设计与样本

- [ ] 样本量是否明确？（物种数、个体数、序列数、细胞数等）
- [ ] 物种或分类群是否具体命名（学名，而非"多种哺乳动物"）？
- [ ] 有无对照组/外群？比较基因组研究有无系统发育背景？
- [ ] 数据来源是否说明（新测序 vs 公共数据库，何种测序平台）？

### 方法可靠性

- [ ] 系统发育分析：模型选择是否合理（碱基替换模型、分子钟校准）？
- [ ] 正选择分析：branch/site 模型，PAML / HyPhy，多重检验校正了吗？
- [ ] 基因组注释：同源映射标准（%identity、coverage），是否有手工验证？
- [ ] 功能验证：是否有体外/体内实验支撑（受体功能、配体结合、表达等），还是纯计算预测？

### 结论边界

- [ ] 因果还是相关？（进化关联 ≠ 功能决定）
- [ ] 收敛演化的判断是否排除了平行演化或祖先多态？
- [ ] 功能预测是否超出数据支持？（如"OR repertoire 扩张→嗅觉更灵敏"是否有行为/生理数据支撑）
- [ ] 统计显著性阈值和置信区间是否汇报？

### 笔记结构

- [ ] `research_question` 是一个明确的问题句，而非标题复述
- [ ] `main_findings` ≤5 条，每条包含具体数据、物种或对象，不能只是泛泛结论
- [ ] `limitations` 只写论文的科学局限，**不能写** AI 数据访问限制
- [ ] `related_concepts` 为英文学科术语（基因名、物种学名、方法名）

---

## 图片规范

笔记只需包含 **1 张代表性图片**（通常是 Figure 1 或最能说明核心结论的图）。

图片来源优先级（由 `build_figures_section` 自动选择）：
1. PMC XML 下载的图片（`pmc-figure-*.png`）
2. 浏览器渲染页面抓取的 HTML 图片（`html-figure-*.png`）
3. PDF 中提取的 Figure 1（`*-fig1.png`）
4. 若全部失败，显示占位文本，不需要手动补充

**不要** 因为"图片不够多"而视为质量缺陷。

## 常见质量问题速查

| 问题 | 典型表现 | 处理方式 |
|------|---------|---------|
| 摘要自报告 | "摘要层面显示…" / "全文未可用" | → 触发 rewrite，move 到 notes 字段 |
| 概念中文化 | `[[嗅觉受体]]` | → 改为英文 `[[Olfactory Receptor]]` |
| 结论过度外推 | "因此 X 进化出了更强的嗅觉" | → 明确写"推测，需功能实验验证" |
| 泛化 main_findings | "研究揭示了基因组演化的规律" | → 补具体物种、基因名、数据量 |
| 样本量缺失 | "多种蝙蝠的比较分析" | → 写"X 种蝙蝠（N 个基因组）" |
