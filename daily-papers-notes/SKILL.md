# Daily Papers Notes

这是 3 步流水线的第 3 步。职责边界要清晰：notes 负责必读笔记、回填和索引，不负责抓取或点评。

## 前置检查

先确认：

- 当日推荐文件存在
- `paper-reader` 可用

如果推荐文件不存在，停止并提示先跑 `daily-papers-review`。

### 浏览器（用于绕过 Cloudflare 等验证）

**默认（无需额外配置）**：脚本自动使用 patchright `launch_persistent_context`，为每个出版商创建独立的持久化 profile（存于系统临时目录，如 `pdf_fetcher_pw_science_org/`）。Cloudflare Turnstile **自动通过**，首次约 10-20 秒，之后因 cf_clearance cookie 复用约 3-5 秒，无需人工操作。

**可选（如需机构订阅 cookies）**：设置 `PAPER_READER_CDP_URL` 指向已登录机构账户的 Chrome 实例：

```powershell
# 先以 remote-debugging 模式启动 Chrome（需关闭现有 Chrome 实例）
Start-Process "chrome.exe" -ArgumentList "--remote-debugging-port=9222","--restore-last-session"
$env:PAPER_READER_CDP_URL = "http://localhost:9222"
```

这样脚本会复用已登录的机构 cookies 尝试访问付费论文全文。

## 读取配置

需要拿到：

- `VAULT_PATH`
- `NOTES_PATH`
- `DAILY_PAPERS_PATH`
- `AUTO_REFRESH_INDEXES`
- `GIT_COMMIT_ENABLED`
- `GIT_PUSH_ENABLED`

## 执行顺序

1. 读取当日推荐文件，只筛出 `必读` 论文。
2. 对每篇必读论文运行：

```powershell
python ..\paper-reader\run_reader.py "{PUBMED_URL_OR_DOI}" --mode standard --prefer-visible-browser
```

> `--prefer-visible-browser` 优先通过 patchright / 浏览器路径尝试获取出版社全文与 PDF。要稳定继承正在使用中的机构 cookies，先用 remote-debugging 启动 Chrome 并设置 `PAPER_READER_CDP_URL`；如果默认 Chrome profile 正在运行且没有 CDP，脚本会退到无 cookies 的临时 profile，并在笔记 Metadata 的 `Cookie source` 中标出。只有在无浏览器、CI 或明确只想跑 PMC/API 摘要模式时，才临时加 `--no-playwright`。

3. 如果已有旧笔记但内容明显不完整，不要直接删除；先重命名为 `*.bak-YYYYMMDD-HHMMSS.md`，再重生成。
4. 运行回填：

```powershell
python ..\daily-papers\backfill_links.py --date YYYY-MM-DD
```

5. 如果 `AUTO_REFRESH_INDEXES=true`，刷新：

```powershell
python ..\_shared\generate_paper_mocs.py
python ..\_shared\generate_concept_mocs.py
```

> MinerU 中间文件由 `paper-reader` 在每篇笔记成功写入后自动清理；正常 notes 流程不需要额外调用清理脚本。`cleanup_runtime_artifacts.py` 仅用于手动清理历史缓存。

## 质量要求

每篇必读笔记生成后至少检查：

- 文件不是空壳
- 有摘要/背景
- 有方法或结果部分

不满足时重新生成。

## Git 提交（按配置执行）

所有必读笔记写入、backfill 完成、MOC 刷新后，如果 `GIT_COMMIT_ENABLED=true`：

```powershell
cd "{VAULT_PATH}"
git add "PaperNotes\" "DailyPapers\YYYY-MM-DD-论文推荐.md"
git commit -m "notes YYYY-MM-DD: N篇必读笔记写入"
```

如果 `GIT_PUSH_ENABLED=true`，追加执行：

```powershell
git push
```

git 操作失败时只打印警告，不中断流程（笔记已写入）。

## 约束

- 只为 `必读` 生成笔记
- 回填脚本也只能给 `必读` 条目插入或替换笔记链接
- 不手写简化版笔记替代 `paper-reader`
- 不生成推荐评论

## HARD GUARDRAILS: must-read note quality

For every must-read note generated by `paper-reader`, verify the final Markdown before considering notes complete:

- It must not contain `?????`, `TODO_AGENT`, `?????`, or equivalent placeholder text.
- `Paper Topic`, `One-Sentence Summary`, `Research Question`, `Main Findings`, and `Core Methods` must contain paper-specific content, not a title echo or generic acquisition-pipeline wording.
- If an existing note is incomplete, back it up before regeneration. Backup notes may remain on disk, but MOC generation must ignore `.bak-` files.
- After regeneration, run `backfill_links.py --date YYYY-MM-DD` and refresh Paper/Concept MOCs.

## HARD GUARDRAILS: must-read PDF/Fig verification

- For each must-read note, verify frontmatter after `paper-reader` finishes. At least one of `downloaded_pdf`, `pdf_path`, or `local_pdf` should point to an existing local PDF for known downloadable publisher DOI prefixes including `10.1111/`, `10.1002/`, `10.1186/`, and `10.1007/`.
- Validate the local PDF by reading its first bytes; it must start with `%PDF`. Do not count a web-text-only note as complete for those DOI prefixes unless a retry with the DOI input also fails.
- If a PubMed URL note has empty PDF fields and a DOI is present, rerun `paper-reader` once with the DOI as the source, then re-check the note.
- Verify any embedded figure path exists. Publisher image files are preferred; PDF-extracted Figure 1 is acceptable as fallback. HTML screenshots are not acceptable as figure success.
- Report PDF/Fig failures explicitly in the final automation summary and memory instead of silently marking notes complete.
